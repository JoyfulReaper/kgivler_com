Random Steam Game Stops library paralysis by choosing a game for you. PUBLIC HEALTHY HTTPS
Container joyful-stack-randomsteam-1
Image joyful/randomsteam
Hostname randomsteam.kgivler.com
Protocol HTTPS / JSON API
Observed memory Approximately 163 MB

A fast random game picker for public Steam libraries. It resolves a Steam account, loads the user's owned games, applies exclusions and filters, and returns a random game without making the visitor wait through a slow picker.

Includes a public website, API endpoints, Steam data caching, rate limiting, and persistent user preferences.

Port TCP 7
Hostname echo.kgivler.com
Role Classic protocol experiment
Observed memory Approximately 40 MB

HappyEcho accepts a TCP connection and sends received data back to the client. The service is intentionally simple: what you send is what you get back.

It exists as a practical networking exercise and as a working example of one of the original small Internet services.

HappyFinger Finger a Steam ID to receive a random game recommendation. PUBLIC TCP :79
Container joyful-stack-happyfinger-1
Port TCP 79
Hostname finger.kgivler.com
Protocol Gopher
Observed memory Approximately 41 MB

HappyFinger is a modern .NET implementation of the classic Finger protocol with a Steam-powered twist. Supply a Steam ID as the Finger username and the service will choose a random game from that public Steam library.

Try it:
finger STEAM_ID@finger.kgivler.com

The Steam profile and owned-games list must be public. This is a real TCP Finger service listening on the protocol's assigned port, not an HTTP simulation.

HappyGopher A real Gopher server for text-first browsing. PUBLIC TCP :70
Container joyful-stack-happygopher-1
Port TCP 70
Hostname gopher.kgivler.com
Protocol Gopher
Observed memory Approximately 36 MB

HappyGopher serves hierarchical menus and text resources using the Gopher protocol. It is a lightweight alternative view of publishing content without the modern web stack.

The result is closer to navigating a remote text filesystem than browsing a typical website.

HappyQOTD Classic Quote of the Day over TCP with a matching HTTP API. PUBLIC TCP :17
Container joyful-stack-happyqotd-1
Port TCP 17
Hostname qotd.kgivler.com
Protocol QOTD / HTTPS JSON API
Observed memory Approximately 36 MB

HappyQOTD serves a single persisted quote for the current UTC day through the classic Quote of the Day protocol on TCP port 17 and through a companion HTTP API.

Try it:
ncat qotd.kgivler.com 17

The TCP service and the API return the same quote of the day, which makes it useful both as a classic protocol demo and as a lightweight integration endpoint.

Mission Control Gateway Receives external events and publishes them internally. SYSTEM HTTP
Container joyful-stack-gateway-1
Loopback port 127.0.0.1:5190
Role Event ingress
Observed memory Approximately 61 MB

The Gateway accepts generic integration events and signed GitHub webhooks. Validated events are wrapped in a common envelope and published to RabbitMQ for downstream processing.

This keeps external HTTP concerns separate from storage, projections, and other event consumers.

Mission Control Archive Maintains a durable SQLite history of integration events. SYSTEM SQLITE
Container joyful-stack-archive-1
Loopback port 127.0.0.1:5191
Storage SQLite named volume
Observed memory Approximately 61 MB

Archive subscribes to the complete Mission Control event stream and writes each event to SQLite. It provides a durable history independent of specialized consumers.

Specialized projections can change without losing the original integration-event record.

Mission Control GitActivity Turns GitHub push events into a website-friendly feed. SYSTEM SQLITE READY JSON API
Container joyful-stack-gitactivity-1
Loopback port 127.0.0.1:5192
Consumes github.push.received
Observed memory Approximately 46 MB

GitActivity consumes GitHub push events, filters them by allowed repositories and branches, and stores a sanitized commit projection in SQLite.

The public website reaches this data through kgivler.api, which keeps the GitActivity service key out of browser code.

RabbitMQ Routes Mission Control events to durable consumers. INTERNAL HEALTHY AMQP
Container joyful-stack-rabbitmq-1
Image rabbitmq:4-management
Virtual host /mission-control
Observed memory Approximately 322 MB

RabbitMQ is the message broker behind Mission Control. A topic exchange routes each integration event to matching durable queues, allowing Archive, GitActivity, and future services to process events independently.

It currently has the largest memory footprint in this stack, but provides durable queueing and reliable separation between publishers and consumers.
